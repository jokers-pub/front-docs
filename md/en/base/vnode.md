## VNode Virtual Node

This section introduces the virtual node, which plays a crucial role in Core and is frequently used in practical logic development.

> Before reading this section, please familiarize yourself with [AST](/base/ast) and [Rendering](/base/render) concepts.

### Overview

A virtual node (VNode) is generated by compiling an abstract syntax tree (AST). It aggregates the current component object and carries the relationships between components as well as node data storage functionality. It not only maps the component's syntactic structure but also implements data encapsulation and management, providing efficient and flexible data support for the entire system's operation.

![VNode](/base/render.png)

Unlike AST, `AST.Node[]` represents structured data of templates in development, while `VNode.Node` represents structured data ready for actual rendering.

For example, a **for** loop:

```xml
@for(let item in 10){
    @item
}
```

At the AST level, the above code contains only two nodes: a `for loop` directive and its child node. This is because AST merely serves as abstract syntax awaiting compilation, not the actual rendering result.  
When rendered, the above code will compile to:

```
├── for loop
│   ├── Text Node 1
│   ├── ...
│   └── Text Node 10
```

It's important to note that `VNode.Node[]` is a complete structured dataset. Regardless of whether a node is to be rendered, it will exist in the virtual structure tree in the form of a node. **Virtual node data is generated by the Core, and manual modification is not recommended.**

As the base type for virtual nodes, `VNode.Node` extends into many functionally significant virtual nodes, including: `VNode.Text`, `VNode.Element`, and more.

`VNode.Node`, as a base type, has the following properties:

| Parameter  | Description                                                                                | Type                   | Default  |
| ---------- | ------------------------------------------------------------------------------------------ | ---------------------- | -------- |
| static     | Whether it's a static node; distinguishes dynamic directives and components for faster updates and rendering | boolean                | -        |
| output     | Output; corresponds to [Render](/base/render). Output varies by platform; for HTML, it's the `DOM Node` | object                 | -        |
| children   | Child nodes                                                                                | VNode.Node[]           | -        |
| parent     | Parent node                                                                                | VNode.Node/undefined   | -        |
| sleep      | Whether the current node is in a sleep state                                               | boolean                | false    |
| prev       | Previous node                                                                              | VNode.Node/undefined   | -        |
| next       | Next node                                                                                  | VNode.Node/undefined   | -        |

Each node comes with predefined properties and a series of API methods for developer convenience.

#### closest

This method allows traversal upward to find the first parent node that meets the specified conditions.

```ts
node.closest<VNode.Element>((node) => {
    return node instanceof VNode.Element;
}, true);
```

-   The first parameter is the query logic function. During traversal, each parent is passed to this function for evaluation. If it returns **true**, the node is identified as the target.
-   The second parameter (`breakWhenVRoot`) restricts the traversal scope. When encountering the root node, traversal stops. Defaults to **false**.

#### find

This method locates matching child nodes and returns all matching children (`VNode.Node[]`).

```ts
node.find((node) => {
    return node instanceof VNode.Element;
});
```

#### contains

This method checks if the current component contains the specified node, returning a **boolean**.

```ts
node.contains((node) => {
    return node instanceof VNode.Element;
});
```

> `VNode.Node` is an entity class instantiated via `new` during rendering. Use `instanceof` to check node types.

### VNode.Root

`VNode.Root` represents the root node of a component—the top-level parent of all nodes in the component. Every component's first node must be an instance of `VNode.Root`.

| Property  | Description      | Type           |
| --------- | ---------------- | -------------- |
| childrens | Child nodes      | VNode.Node[]   |
| component | Component instance | IComponent     |

### VNode.Text

`VNode.Text` represents a text node.

| Property | Description  | Type   |
| -------- | ------------ | ------ |
| text     | Text content | string |

### VNode.Html

`VNode.Html` serves as an HTML node, where content blocks are rendered as HTML.

| Property | Description    | Type   |
| -------- | -------------- | ------ |
| html     | HTML content   | string |

### VNode.Comment

`VNode.Comment` serves as a comment node, outputting comment content during rendering.

| Property | Description   | Type   |
| -------- | ------------- | ------ |
| text     | Comment text  | string |

### VNode.Element

`VNode.Element` is a generic tag node. Non-component elements are rendered as `VNode.Element` nodes.

| Property    | Description      | Type                                                                                                      |
| ----------- | ---------------- | --------------------------------------------------------------------------------------------------------- |
| attributes  | Tag attributes   | Record<string, any>                                                                                       |
| events      | Node events      | Array<[string, { modifiers?: string[]; callBack: EventCallBack }]> `EventCallBack` (see **VNode.Event**) |
| tagName     | Tag name         | string                                                                                                    |
| childrens   | Child nodes      | VNode.Node[]                                                                                              |

### VNode.Component

`VNode.Component` is a component node used for component data processing and as a container for component mounting.

| Property       | Description                                                                                     | Type                                                                                                      |
| -------------- | ------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
| name           | Component name                                                                                   | string/undefined (for dynamic components)                                                                 |
| component      | Component instance                                                                               | Component                                                                                                 |
| events         | Component events                                                                                 | Array<[string, { modifiers?: string[]; callBack: EventCallBack }]> `EventCallBack` (see **VNode.Event**) |
| propValues     | Component parameters                                                                             | Record<string, any>                                                                                       |
| keepalive      | Whether the component should maintain state (corresponding to the **keep-alve** property in [template built-ins](/base/template-property)) | boolean                                                                                                   |
| firstElement   | The first `VNode.Element` node inside the component                                              | VNode.Element/undefined                                                                                   |
| childrens      | Child nodes                                                                                      | VNode.Node[]                                                                                              |

Developers often use this type to implement complex requirements, such as:

```ts
class extends Component{
    get parentComponent(){
        this.$rootVNode.closest((n)=>{
            return n instanceof VNode.Component && n.component.name='my-component-id';
        })?.component;
    }

    test(){
        // Call parent component API
        this.parentComponent?.testFunction();
    }
}
```

> The **output** property of `VNode.Component` varies across rendering platforms. For example, in HTML-Render, it outputs `document.createTextNode("")`—a blank text node marking the node's position.

### VNode.Condition

`VNode.Condition` represents a conditional node, implementing the `if`/`elseif`/`else` directives in templates.

| Property  | Description                                                                                                                                                     | Type                |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------- |
| cmdName   | Directive type                                                                                                                                                 | `if`/`elseif`/`else` |
| result    | Node condition evaluation result (non-deterministic). During `if-else` evaluations, if a prior condition is `true`, subsequent conditions won't update until the prior evaluates to `false`. | boolean             |
| isShow    | Whether the child nodes are rendered (reflects actual **rendering result**, not evaluation result).                                                           | boolean             |
| childrens | Child nodes                                                                                                                                                    | VNode.Node[]        |

### VNode.List / VNode.ListItem

`VNode.List` is a list-rendering node used for handling loop data and providing a container for list items.

| Property  | Description | Type             |
| --------- | ----------- | ---------------- |
| childrens | Child nodes | VNode.ListItem[] |

`VNode.ListItem` is an item-rendering node within a list, serving as a container for content nodes.

| Property  | Description                                                                 | Type         |
| --------- | --------------------------------------------------------------------------- | ------------ |
| ob        | Data object for the current item, including loop-configured `indexKey` and `itemKey` | Component    |
| childrens | Child nodes                                                                | VNode.Node[] |

### VNode.RenderSection

`VNode.RenderSection` is a block-rendering node used for rendering **templates awaiting rendering** within components.

| Property | Description         | Type                     |
| -------- | ------------------- | ------------------------ |
| id       | Block name          | string (default: unknown) |
| params   | Block parameters    | any[]                    |
| section  | Block-rendering object | SectionType             |

The `SectionType` includes:

| Property | Description                                                                                           | Type       |
| -------- | ----------------------------------------------------------------------------------------------------- | ---------- |
| asts     | AST tree (see [AST](/base/ast))                                                                      | AST.Node[] |
| ob       | Data object (`ob`) needed for rendering this template                                               | Object     |
| parser   | Parser object context for rendering this template                                                   | ParserTemplate |
| params   | Parameters (see the **block parameters** section in [Block Rendering](/base/template-section)). Represents **parameter names**, not values. | string[]   |

### VNode.Event

`VNode.Event` serves as the event-handling object in Joker's frontend. Whether triggered via JS or native events, this type is passed as the first argument to event handlers.

Its `data` property represents event parameters and can be type-specified via generics for ease of use (defaults to `undefined`).

| Property          | Description                     | Type                 |
| ----------------- | ------------------------------- | -------------------- |
| eventName         | Event name                      | string               |
| event             | Native event object (platform-dependent) | any                  |
| target            | Virtual node triggering the event | VNode.Node/undefined |
| data              | Event parameters                | <T>: any             |
| preventDefault    | Prevent default event behavior   | function             |
| stopPropagation   | Stop event propagation          | function             |

Event handler functions (`EventCallBack`) must accept **VNode.Event as their first parameter**.

```ts
export type EventCallBack<T = any> = (e: VNode.Event<T>) => void;
```

> For event handling, refer to the [Component API's](/base/component-api) `$on` method and [Template Event Registration](/base/template-event).
